# TaskMatrix — AI Agent Guide

Instructions for Hermes, Claude Code, Codex, and any other AI coding assistant working on this project.

## What Is This

TaskMatrix is a single-file vanilla JS PWA implementing the Eisenhower Matrix. You add tasks by typing or voice, they get classified into quadrants, and everything syncs to Supabase. No frameworks, no build step — open `index.html` in a browser or deploy to GitHub Pages.

**Repo:** `ali999774.github.io/TaskMatrix_Supabase` (GH Pages)
**Live:** https://ali999774.github.io/TaskMatrix_Supabase
**Local dev:** `python3 -m http.server 8000` → http://localhost:8000

## Stack

- **Vanilla JS** — no React, no Vue, no TypeScript. Single `index.html` (~197KB) with inline `<script>` blocks + separate `src/*.js` files
- **Supabase** (PostgreSQL + Realtime) for sync and auth
- **Dexie.js** (IndexedDB) for offline cache
- **Google OAuth** via Supabase Auth
- **GitHub Pages** — push to `main` → auto-deploy in ~60s
- **Service worker** (`sw.js`, cache version `taskmatrix-v4`) — PWA offline support
- **WebLLM** (`@mlc-ai/web-llm` via CDN import) — browser-side LLM inference
- **Playwright + Jest** for tests

## API Keys (IMPORTANT)

**Keys live in browser localStorage, never in committed files.**

- `src/config.js` — safe to commit, contains NO secrets
- `src/keys-ci.js` — **generated by CI at deploy time** from GitHub Secrets, NEVER committed (gitignored)
- Set keys locally via browser console: `_tmSetApiKey('ANTHROPIC_API_KEY', 'sk-ant-...')`
- Keys persist in localStorage under `tm_key:*` prefix
- Ali had API keys leaked on GitHub before — DO NOT add keys to any committed file. If you need to reference a key, use the `_tmSetApiKey()` pattern.

## Model Router (3-Tier)

File: `src/model-router.js`. Single `window.modelRouter` instance. App never knows which tier is active.

| Tier | Backend | Default Model | When It's Used |
|------|---------|---------------|----------------|
| Edge | WebLLM (browser GPU) | Gemma 3 1B | Offline-first, no API key needed |
| Cloud | Anthropic / OpenAI | Claude Sonnet 4 / GPT-4o | Best quality, requires key |
| Local | Ollama | qwen2.5:7b | Self-hosted, privacy |

- Config: `window.MODEL_ROUTER_CONFIG` (defined in `src/config.js`)
- Fallback chain: cloud → edge → local (configurable)
- Runtime tier switching via `modelRouter.switchTier('cloud')`
- Edge tier caches model in Cache API after first download (`cacheModel: true`)
- All keys read from `window.ANTHROPIC_API_KEY`, `window.OPENAI_API_KEY`, etc.

## Database

- Table `tasks` — main task store
- Table `sticky_notes` — pinned notes (columns: id, user_id, content, title, color, position_x, position_y, pinned, created_at, updated_at)
- Task IDs: `id` = ephemeral local float (`Date.now() + Math.random()`), `supabaseId` = stable UUID. Always use `supabaseId` for DB operations.

### Critical DB Rules (violating these caused production bugs)

- **Upsert:** Must include `id` in payload for conflict resolution
- **Update:** Must NOT include `id` in payload — Postgres rejects PK updates. Destructure it out before `.update()`
- **UUID in onclick:** Must be quoted: `onclick="fn('${id}')"` NOT `onclick="fn(${id})"` — hyphens in UUIDs cause JS arithmetic evaluation → NaN

## Known Issues

- **Sync dot stuck red/orange** — FIXED (2026-05-18). Added 30s heartbeat (`_startSyncHeartbeat`) that auto-heals stale `_syncHasError` and `_syncRealtimeOk` flags. Starts on auth, stops on sign-out.
- **Shell-init garbage** — MITIGATED. CI deploy workflow now runs a sanitizer step that strips `shell-init:` and `chdir:` lines from all deployed files. Local: strip with `re.sub(r'^(shell-init:|chdir:).*\n?', '', content, flags=re.MULTILINE).lstrip('\n')`

## Architecture Gotchas

- **All modals must be direct children of `<body>`** — never nested inside other modals or containers with `display:none`. This caused multiple invisible-modal bugs.
- **`mapRowToTask`** — Supabase reload generates new random IDs. Fix: reuse existing in-memory ID: `tasks.find(t => t.supabaseId === row.id)?.id ?? (Date.now() + Math.random())`
- **Touch ghost clicks** — deferred synthetic clicks from touch events can fire on modal overlays. Guard: 300ms timestamp check on `closeModalOnOverlay`.
- **Service worker cache** — version `taskmatrix-v4`. Always hard-refresh after deploy.
